# よくある質問

## 実験参加について

### メディア実証実験に参加するには WordPress またはそれをベースとした CMS を使う必要がありますか

いいえ。WordPress のプラグインはあくまでも参照実装であり、各社で実際にご利用の CMS に OP 対応を行って実験に参加頂けます。

### メディア実証実験に参加するには CMS の内部実装に大きく手を入れる事が必須でしょうか

いいえ。WordPress の参照実装のように、CMS 内部の処理で SDP を生成・登録する形にすることが望ましいですが、CMS 内部の実装に手を入れるのが難しい場合、静的 HTML サイトと `website.json` をインプットとして CLI で SDP を作成するように、CMS が出力した結果の HTML に対して外部の CLI などで SDP を生成・登録し、CMS 側には登録された Profile Set を返すための link タグを入れるだけで済ませられる可能性はあります。

但し、このような形式で対応するには、公開前の段階で公開時と全く同じ HTML に CLI からアクセス出来ること、抜き出す対象要素を特定可能である (id や class が事前に分かる) などの前提条件があります。

### 複数のサイトを運用しているのですが、どのサイトで試験するのが良いでしょうか

自社コンテンツを掲載するサイトであればいずれでも構いません。単純な構成のサイトの方が対応は簡単ですが、実装上の課題を見つけるという目的としては、より複雑な仕組みのサイトで実験頂く方が良いかも知れません。本ドキュメントをご覧になって可能な範囲で実験頂くとか、順次実験頂くことをご検討ください。

### 公開鍵・プライベート鍵を作成する方法を教えてください

[鍵ペアの作成](./registry/operation/key-pair-generation.md)を参照してください。

### 実験で OP CIP 提供の OP/DP レジストリの API を使ったところ、 400 Bad Request のエラーが返ってきました。どのような原因が考えられますか

[CIP 提供 DP レジストリについて](./media-study-202307/howto.md#cip-提供-dp-レジストリについて)をご確認ください。それでも解決しない場合はお問い合わせください。

## OP/DP の使い方・運用について

### OP ID とは何ですか。コンテンツを出版するサイトのドメイン名と OP ID は一致させたほうがよいですか

OP ID は法人などの組織に対して割り振られる ID です。 ID の形式として、その組織の情報を示す主たるサイトのドメインを採用しています。コンテンツを出版するサイトのドメイン名と OP ID は異なるもので構いません。1組織が複数のメディアサイトを運営する場合や、メディアサイトとは別に企業ページがある場合には、一致させないほうがよいでしょう。

### 利用者の課金状態などに応じて表示が変わるコンテンツにはどのように DP を発行すべきですか

課金・無課金でのコンテンツそれぞれに対して DP を発行してください。詳細は[記事管理に関するケース毎の対応方法](./media-study-202307/sdp-case-study.md)を参照してください。

### 複数ページにわたるコンテンツにはどのように DP を発行すべきですか

各ページに1つずつ DP を発行してください。詳細は[記事管理に関するケース毎の対応方法](./media-study-202307/sdp-case-study.md)を参照してください。

### コンテンツを更新した際には、必ず DP の再発行が必要ですか

更新内容によります。更新により DP に記載した情報が古くなった場合には再発行を実施してください。例えば発行した DP の中に `https://schema.org/dateModified` プロパティがある場合には、更新日時の変更のために DP を再発行する必要があるでしょう。

### コンテンツを更新し、それに合わせて DP も再発行したいです。 出版物の ID（DP の `sub` クレームの値）は、更新前後で同じ値にしたほうがいいですか

同じ値にしてください。OP CIP 提供の DP Store (dprexpt.originator-profile.org) では、同じ出版物 ID を持つ DP が複数 DP Store に登録されている場合には、その中の最新の1件を API のユーザーに返します。

### 記事を非公開・削除しました。 DP も削除すべきでしょうか

削除したほうがよいでしょう。 DP の中にはタイトルや著者などの記事の情報が含まれているため、記事削除・非公開後に DP にアクセスできる場合、それらの情報がユーザーに見えてしまいます。

### 記事を非公開状態で新規作成しました。 DP は発行すべきですか

DP は記事の公開と同時に発行され、記事閲覧ユーザーからアクセスできることが期待されます。非公開状態での新規作成の場合は DP は発行しなくても構いません。

### DP に履歴の概念はありますか

OP CIP 提供の DP Store (dprexpt.originator-profile.org) では、保存されている DP が更新された際に、古い DP も DB に残してあります。しかし、 DP のデータには、 DP の更新履歴やバージョンについての情報はありません。ユーザーには最新の DP の情報だけが提示されます。
