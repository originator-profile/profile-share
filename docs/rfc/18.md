## SD-JWT VC Type Metadata

[SD-JWT-based Verifiable Credentials (SD-JWT VC)](https://www.ietf.org/archive/id/draft-ietf-oauth-sd-jwt-vc-04.html) を参照してください。

## JWT VC Issuer Metadata

[SD-JWT-based Verifiable Credentials (SD-JWT VC)](https://www.ietf.org/archive/id/draft-ietf-oauth-sd-jwt-vc-04.html) を参照してください。

## Originator Profile ID (OP ID)

Originator Profile RFC 文書で定義される組織情報の識別子

## Originator Profile (OP)

Originator Profile RFC 文書で定義される SD-JWT VC 形式による基本的な組織に関する情報

## OP 保有組織

OP の `sub` クレームが指す OP ID に紐づく組織。

## OP レジストリ

OP の発行者。

## 検証者

OP が定義するデータを受け取って検証する人・組織。例えばインターネット上のコンテンツを閲覧するエンドユーザーや、検索エンジンクローラーなどのボットが挙げられる。

## Web Content

Web ページ上の複数 DOM で表現された、情報として一つのまとまりを持つ要素。1 つの Web Content は単一の組織によって出版される。 DOM 上で1つの要素に収まるとは限らない。

## Content Assertion (CA)

Content Assertion RFC 文書で定義される Web Content に関する情報

## CA ID

CA の `sub` クレームに指定される ID。 Web Content と1対1対応する。

## Content Assertion Set (CAS)

Content Assertion Set RFC 文書で定義される複数の Content Assertion を提示するための表現形式

## Target Integrity

Web Content の完全性を保証する暗号論的ハッシュを持ち、Web ページ上での位置および範囲を特定するデータ

## 要素位置

Target Integrity で特定される Web Content の位置。

## 署名対象コンテンツ

Target Integrity で完全性が保証される Web Content の文字列表現。

## Certificate

Certificate RFC 文書で定義される SD-JWT VC 形式により、発行者が OP 保有組織に対して内容を記述したことを証明する情報。証明書。

## 認証機関 (Certifier)

認証制度 (Ceritification System) を運用する主体。Certificate の発行者とは限らない。
