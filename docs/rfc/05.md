# HTML への埋め込み - Editor's Draft

## 概要

本文書では、 Web Assertion Set を HTML 文書上で提示する方法を説明します。

## 用語

本文書に説明のない用語については、用語 RFC 文書を参照してください。

- Web Assertion (WA)
- Web Assertion Set (WAS)

## 埋め込み表現の形式

script 要素を使用して、要素の本体が Web Assertion Set であることを示します。

スクリプトの種類が Web Assertion Set であることを示すため、 type 属性には本仕様で定義するメディアタイプを使用しなければなりません (MUST)。

### インライン

script 要素のコンテンツに Web Assertion Set を記述します。

#### 例

単一の WA を提示した WAS を埋め込む具体例を次に示します。

```htmlembedded
<script type="application/was+json">
{
  "originator": "eyJ...~",
  "certificates": ["eyJ...~"],
  "assertions": ["eyJ...~"]
}
</script>
```

複数の WA を提示した WAS を埋め込む具体例を次に示します。

```htmlembedded
<script type="application/was+json">
[
  {
    "originator": "eyJ...~",
    "certificates": ["eyJ...~"],
    "assertions": ["eyJ...~"],
  },
  {
    "originator": "eyJ...~",
    "certificates": ["eyJ...~"],
    "assertions": ["eyJ...~"]
  },
  ...
]
</script>
```

### 外部参照

src 属性に Web Assertion Set の URL を記述します。

integrity 属性には読み込むファイルのハッシュ値を指定します。
ハッシュ値はサブリソース整合性 (SRI) の検証に使用します。
ハッシュ値の形式は [SRI セクション 3.1](https://www.w3.org/TR/SRI/#framework) の Integrity metadata でなければなりません (MUST)。

#### 例

URL で WAS を埋め込む具体例を次に示します。

```htmlembedded
<script type="application/was+json" src="https://example.com/was.json" integrity="sha256-XnUoFByIs5DIz6wAvte7AfpYeqPrs42KLR1Mlg9+A/M="></script>
```

## メディアタイプ

Web Assertion Set に使用するメディアタイプは `application/was+json` です。
