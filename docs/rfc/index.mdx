# Originator Profile RFC

_この文書は非規範的です。_

Originator Profile の技術仕様の文書化を目指します。

これらの文書では、Originator Profile の技術的側面について説明しています。
Originator Profile 技術は、特定のコンテンツの善悪の価値判断を提供するものではなく、暗号的に検証可能な手段を提供することで、誰もが信頼性を評価できるようになることを目指しています。

:::note
これらの文書はすべて<ruby>草案<rt>draft</rt></ruby>です。
:::

OP の仕様は VC とその配布方法を定義します。
仕様全体で共通の用語の定義が[用語集](./terminology.md)にあります。

OP は VC 共通の仕様として次の仕様を定めます。

- [OP VC Data Model](./op-vc-data-model.md): OP で定める VC が準拠するデータモデル
- [Securing Mechanism](./securing-mechanism.md): OP で定める VC の Securing Mechanism

組織の ID として次を定義し、各 VC で利用します。

- [OP ID](./op-id.md): OP ID
- [DNS URI OP ID](./dns-uri-op-id.md): ドメイン名を利用した OP ID

中心的な役割を果たす3つの VC のデータモデルを定めた文書が次になります。

- [Core Profile](./cp.md): 組織の OP ID と公開鍵の VC
- [Profile Annotation](./pa.md): 組織に関する情報を記述する VC。情報に応じて適切な主体が署名する。
- [Content Attestation](./ca.md): OP が作成や管理する Web Content などに関する情報や対象コンテンツの特定、検証方法などを記載し CP 保有者が署名するデータ

他に次の VC も定義します。

- [Web Media Profile](./web-media-profile.md): Web での OP の利用においてユーザーに表示する情報を記述する VC。 CP 発行者が署名する。
- [Website Profile](./website-profile.md): Web サイトを保有していることを示す VC。 CP 保有者が署名する。

[Profile Annotation](./pa.md), [Content Attestation](./ca.md) を実際に利用する際にはアプリケーションに応じて具体的なプロパティを定義する必要があります。
OP CIP で利用するプロパティ定義は次の実装者ガイドで定めています。

- Content Attestation
  - [Article Data Model](./ca-guide/article.md): インターネット上の記事に紐づく CA のデータモデル。
  - [Online Ad Data Model](./ca-guide/online-ad.md): インターネット広告に紐づく CA のデータモデル。
- Profile Annotataion
  - [Certificate Data Model](./certificate.md): 証明書 PA のデータモデル。
    - [Existence Certificate Properties](./pa-guide/existence.md): 実在性確認証明書 PA のデータモデル。
    - [JICDAQ 認証 Certificate Properties](./pa-guide/jicdaq.md): JICDAQ 証明書のデータモデル。
    - [日本新聞協会所属 Certificate Properties](./pa-guide/japan-pressnet.md): 日本新聞協会所属証明書のデータモデル。

これら VC をまとめて配布する際のデータ形式として次のものを定義します。

- [Content Attestation Set](./content-attestation-set.md): CA の配布形式
- [Originator Profile Set](./originator-profile-set.md): CP, PA, Web Media Profile の配布形式
- [Site Profile](./site-profile.md): Website Profile の配布形式
- [Linking](./link-to-html.md): CAS, OPS を HTML ページに紐づける方法

CA にはコンテンツの改ざんを防ぐ仕組みとして Target Integrity があります。次の実装ガイドがそれらの仕様です。

- [Target Integrity Registry](./target-guide/index.mdx): Target Integrity のレジストリ
- [HTML Target](./target-guide/html.md): 対象 DOM 要素の HTML 文字列としての完全性検証
- [Text Target](./target-guide/text.md): 対象 DOM 要素に含まれるテキストコンテンツの完全性検証
- [Visible Text Target](./target-guide/visible-text.md): 対象要素のレンダリングされているテキストの完全性検証
- [External Resource Target](./target-guide/external-resource.md): 外部リソースの完全性検証

OP の枠組み全体のセキュリティを担保する文書として次のものがあります。

- [暗号アルゴリズム](./algorithm.md): OP で推奨される暗号アルゴリズム

VC 全体で使われる JSON-LD のコンテキストは次の文書に記載しています。
