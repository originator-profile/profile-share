# よくある質問

- Q. 本実験に参加するには WordPress またはそれをベースとした CMS を使う必要がありますか。
  - いいえ。WordPress のプラグインはあくまでも参照実装であり、各社で実際にご利用の CMS に OP 対応を行って実験に参加頂けます。
- Q. 本実験に参加するには CMS の内部実装に大きく手を入れる事が必須でしょうか。
  - いいえ。WordPress の参照実装のように、CMS 内部の処理で SDP を生成・登録する形にすることが望ましいですが、CMS 内部の実装に手を入れるのが難しい場合、静的 HTML サイトと `website.json` をインプットとして CLI で SDP を作成するように、CMS が出力した結果の HTML に対して外部の CLI などで SDP を生成・登録し、CMS 側には登録されたプロファイルセットを返すための link タグを入れるだけで済ませられる可能性はあります。
    但し、このような形式で対応するには、公開前の段階で公開時と全く同じ HTML に CLI からアクセス出来ること、抜き出す対象要素を特定可能である (id や class が事前に分かる) などの前提条件があります。
- Q. 複数のサイトを運用しているのですが、どのサイトで試験するのが良いでしょうか。
  - 自社コンテンツを掲載するサイトであればいずれでも構いません。単純な構成のサイトの方が対応は簡単ですが、実装上の課題を見つけるという目的としては、より複雑な仕組みのサイトで実験頂く方が良いかも知れません。本ドキュメントをご覧になって可能な範囲で実験頂くとか、順次実験頂くことをご検討ください。
