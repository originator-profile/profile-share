# 実装状況と制限事項

このページでは、開発チームによる実装状況の共有とその状況による制限事項について説明します。

:::note
開発チームによるリリースに応じて、このページの記載内容は更新されます。
:::

## 実証実験全体

- オンライン広告 (昨年度実験) の CA には未対応

## Extension

- SP, CAS, OPS の取得と検証処理は実装しているが、デザインされた UI での表示が未実装
  - GitHub Issue: [WebMediaProfile (WMP) が OP に含まれていない場合に利用者の SP / OP / CA の閲覧性を改善する · Issue #1890 · originator-profile/profile](https://github.com/originator-profile/profile/issues/1890)
- 設置先ドメイン制限など、一部の検証処理に未対応
  - GitHub Issue: [Website Profile のオリジン一致性確認による検証処理の実装 · Issue #1685 · originator-profile/profile](https://github.com/originator-profile/profile/issues/1685)

## CLI

- 旧仕様の名称や機能が混在
  - 将来修正予定としています。

## WordPressプラグイン

- OPS リンクの自動挿入に未対応
  - サイト運営者 = コンテンツ作成者の場合は不要です。
  - 記事毎の作成組織が WordPress 全体の運営者と異なるケースは WordPressプラグイン としては「現状」対応しません。
- CAS の WP 側の保存と integrity 属性付与に未対応

## CA Server

- Site Profile 生成 API は検討中の仮実装
- これは本来の仕様通りですが、サーバとしては CORS 対応する可能性もあります。

## OP 登録サイト

- 登録サイトのフォームでは必要な情報の過不足あり
  - お手数ですが、別途 GitHub Issues 等でのやり取りさせていただきます。
  - 最新のデータモデル仕様で必要としている入力項目や記載内容の制限などに未対応
    - 例：情報発信ポリシー周りの項目
- 同梱すべき証明書 (PA；サイトの UI 上では「資格情報」という古い用語となっています) の選択フォームが動作不可
  - 「未登録」のまま登録＆申請を進めてください。
- 公開鍵の登録フォームが動作不可
  - CA Serverをご利用の場合は、鍵の生成および管理をCA Server上で行うためここでの公開鍵の登録は不要です。「未登録」のまま登録＆申請を進めてください。
  - CA Serverをご利用されない場合、公開鍵の登録が必要となりますが、登録フォーム上は「未登録」のまま登録＆申請を進め、別途公開鍵を送付ください。

:::note
2024年度は開発を凍結しており、年度初め時点の実装をそのまま利用していることによる制約です。
:::
