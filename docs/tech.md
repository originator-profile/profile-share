# OP 技術について

## OP 技術の新規性

OP 技術は、インターネット上のコンテンツに発信者の実在性、信頼性情報を紐づける技術です。それらの情報を Originator Profile (OP)、 Document Profile (DP) というデジタル署名付きのデータとしてコンテンツと共に提供することで、ユーザーがコンテンツやその発信者の正しい情報を閲覧し、コンテンツの信頼性を判断できることを目指します。

_（箇条書きを文章にする）_

- 既存の Web コンテンツに対して検証範囲を指定して改ざん防止の署名を付与できる
- IHV モデル / VC の利用によって検証時のサーバアクセス不要、プライバシーも守られる
- ID にデフォルトで信頼性を表す情報として資格情報が同梱されるデータモデル (X.509 ではできない)
- 組織の実在性確認や資格情報の質、あるいは(コンテンツ情報の信頼性も)担保するガバナンス設計
- インターネットと同様に分散型のアーキテクチャで特定のレジストリ等運営団体に依存しない (インターネットの常識)

_（ここから他の技術との比較）_

従来のインターネットでは、ユーザーが、閲覧しているコンテンツがどのような組織によって作成され流通しているのかを確認することができませんでした。

コンテンツよりも粗い粒度での確認に関しては、EV証明書が利用できるでしょう。
Web サイトのドメインを保有している法人の確認には、 EV証明書が利用できます。 EV証明書は発行の際に証明書取得者の本人性を確認することが義務付けられています。EV証明書が設置されている Web サイトを訪れたユーザーは、ブラウザのアドレスバーから証明書の情報を確認することで、ドメイン保有者の実在性が確認できました。
一方で、 EV証明書はユーザーの被害を防止するためには効果が薄く、ブラウザベンダーの支持を得られなくなりました。

## データモデルの技術的な特徴

- 組織の実在性、信頼性を表す OP と、コンテンツの信頼性を表す DP とがある
- OP, DP は JWT である
- OP, DP はレジストリ、あるいはコンテンツ発行者のプライベート鍵で署名がなされている
- OP には組織の基本情報と資格情報が含まれる
- DP にはコンテンツの一部について改ざんを検出するための署名（あるいはハッシュ）が含まれる
- DP にはコンテンツに関する付加情報が含まれる

これらを実現するために、 OP 技術では、組織に対して Originator Profile (OP)、コンテンツに対して Document Profile (DP) を発行します。これらは発行団体の持つプライベート鍵で署名がされています。
Originator Profile (OP) は JWT になっています。この中に、組織名、住所、電話番号などの組織に関する情報が入ります。また、第三者機関による資格情報も OP の中に含まれています。これら組織の情報、資格情報の正しさは、 OP を発行する前に人による審査によって検証されます。 OP は、 OP レジストリのプライベート鍵で署名がされます。

Document Profile (DP) は JWT になっています。 DP は OP 保有法人の持つプライベート鍵で署名がされています。OP 保有法人は、コンテンツを出版する際にそのコンテンツに関する情報をまとめて仕様で定められた DP のフォーマットにし、プライベート鍵で署名します。プライベート鍵に対応する公開鍵は、 OP の中に含まれます（あるいは特定の URL に取りに行く）。

OP, DP はコンテンツをインターネット上で公開する際に、 HTML の中に書きます（ link 要素、 script 要素など）。
ユーザーは、コンテンツ閲覧時に、 OP 拡張機能を開くことで、コンテンツに紐づく OP, DP を確認することができます。
それらの検証に失敗した場合には、検証失敗を示す画面が表示されます（スクリーンショット）。
検証に成功した場合には、 OP, DP 内の情報がユーザーに表示されます（画像）。ページ内の各コンテンツの位置も特定されます。

## 一般利用者が得られる（技術的な観点からみた）恩恵

OP/DP が存在するページでは

- OP, DP に付与された署名を検証することによってそれらの情報の真正性が担保されます。これは OGP を利用してメタデータをつけることでは実現できません。
- コンテンツが改ざんされていないかどうかをチェックすることができます
- 組織の保有する資格情報を確認することで組織の信頼性を確認できます
